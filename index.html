Usability Evaluation and Redesign of eCitizen Government Service Portal

This project focuses on evaluating the usability of the eCitizen government service portal and proposing a user-centered redesign to improve accessibility, efficiency, and user satisfaction for Kenyan citizens interacting with government services online.

Problem Statement

Complex forms with technical language create barriers for users

User frustration leads to low adoption of digital services

Inconsistent interfaces across different services

Although eCitizen centralizes many government services, users often struggle to complete applications successfully. The system presents long forms, technical language, and inconsistent layouts across services, which increases user errors and frustration.

Target Users

The primary users are Kenyan citizens with varying levels of digital literacy who access the portal using mobile phones and desktop computers. Secondary users include government officers who review and process submitted applications.

Existing System Analysis

The current eCitizen system provides centralized access but suffers from inconsistent user interfaces, unclear navigation paths, limited feedback during errors, and poor mobile optimization.

Design Approach

The project follows a user-centered design approach guided by established HCI principles. User needs, task requirements, and usability challenges informed the design decisions with focus on simplifying navigation, reducing cognitive load, and providing clear step-by-step guidance.

Design Alternatives

Form-Based Design

Traditional form layout with all fields displayed at once. Provides complete overview but can be overwhelming for complex applications.

Wizard-Based Design

Step-by-step approach breaking complex forms into smaller sections. Reduces cognitive load but may hide overall progress.

Guided Assistant Design

Interactive assistant that guides users through the process with contextual help and explanations. Most user-friendly but complex to implement.

Final Design Overview

The final design presents a simplified and consistent interface for government service applications. It introduces a unified layout, clear progress indicators, and structured application steps that guide users from service selection to successful submission.

Evaluation & Results

Usability testing with participants

Analysis of evaluation data

Evaluation Plan

The evaluation plan included usability testing with representative users, heuristic evaluation based on HCI principles, and a user questionnaire to collect subjective feedback on the system's usability.

Evaluation Results

What Worked Well

  • Step-by-step guidance improved task completion
  • Consistent layout reduced confusion
  • Progress indicators helped users understand their position
  • Structured application flow reduced errors

Areas Needing Improvement

  • Document upload requirements were unclear
  • Error messages needed more detail
  • Contextual help was insufficient
  • Mobile optimization could be enhanced

User Feedback

User feedback indicated that the step-by-step guidance and consistent layout made the system easier to use. However, some users expressed confusion regarding document upload requirements and suggested clearer instructions and error messages to improve the experience.

Suggested Improvements

If more time were available, the design could be improved by adding clearer document upload guidance, more informative error messages, enhanced accessibility features, and multilingual support to better serve a wider range of users.
