Initial Problem Statement

Although eCitizen centralizes many government services, users often struggle to complete applications successfully. The system presents long forms, technical language, and inconsistent layouts across services, which increases user errors and frustration. These usability challenges reduce trust in digital government services and discourage adoption.

Comprehensive User & Task Analysis

Our research revealed critical insights into the diverse user base and their interaction patterns with the eCitizen portal. We identified three primary user personas and their specific needs, tasks, and pain points.

Detailed Task Analysis

We mapped the complete user journey for common eCitizen services:

Service Selection Phase
  • Navigate through hierarchical service categories
  • Read service descriptions and requirements
  • Determine eligibility and prerequisites
Application Phase
  • Fill personal information forms
  • Complete service-specific sections
  • Review information before submission
Post-Submission Phase
  • Receive confirmation and reference number
  • Check application status periodically
  • Respond to any additional requests

Key Pain Points Identified

Through user interviews and usability testing, we identified critical issues:

  • Cognitive Overload: Forms with 50+ fields displayed at once
  • Inconsistent Design: Different layouts for similar services
  • Poor Error Feedback: Generic messages without guidance
  • Mobile Unfriendly: Tiny touch targets, poor responsiveness
  • Technical Jargon: Government terminology without explanations
  • Unclear Navigation: Users get lost in complex menu structures
  • Document Confusion: Unclear requirements for uploads
Impact:

These issues lead to high abandonment rates, increased support calls, and low user satisfaction scores averaging 2.8/5 across services.

Existing System Analysis

The current eCitizen system provides centralized access to government services and reduces the need for physical visits. However, it suffers from inconsistent user interfaces, unclear navigation paths, limited feedback during errors, and poor mobile optimization. These issues significantly affect task completion and overall user satisfaction.

Key Technical Constraints Identified

Backend Integration

Must connect with multiple government agency databases with varying APIs

Security Requirements

Must comply with government security standards for citizen data protection

Accessibility Standards

Must meet WCAG 2.1 AA standards for users with disabilities

Design Alternatives: Comprehensive Analysis

We explored three distinct design alternatives to address the core usability issues identified in our research. Each was chosen to test a different hypothesis about how to best support users with varying digital literacy.

1. Form-Based Design

Why Chosen: This alternative was selected as a baseline, replicating the existing system's structure but with improved visual organization. It tests whether clarity can be achieved through better layout alone, without changing the fundamental interaction model.
Usability & Appropriateness:

It is appropriate for experienced or task-focused users who value speed and have a clear mental model of the required information. Its usability strength is providing a complete overview, allowing users to skip around and fill known fields quickly.

Unique Elements (Not in Other Designs):
  • All input fields visible simultaneously for complete context
  • Supports non-linear form completion (skip to any section)
  • Traditional layout familiar to most digital users
  • Easy to print or save entire form at once
Appropriate For:

Simple applications with fewer than 15 fields, users with high digital literacy, scenarios where users need to reference information from different sections simultaneously.

2. Wizard-Based Design

Why Chosen: This alternative was chosen to directly combat cognitive overload by decomposing long, complex applications (like business licenses or passport renewals) into a sequence of simple, manageable steps.
Usability & Appropriateness:

It is highly appropriate for procedural or lengthy tasks and for novice or uncertain users. Its key usability feature is reducing errors and anxiety by presenting one logical piece of the task at a time, with clear "Next" and "Back" controls.

Unique Elements (Not in Other Designs):
  • Visual progress indicator showing completion status
  • Contextual help specific to each step
  • Logical grouping of related fields into thematic steps
  • Automatic validation before proceeding to next step
Appropriate For:

Complex applications with 15+ fields, first-time users, processes with clear sequential dependencies, mobile interfaces where screen space is limited.

3. Guided Assistant Design

Why Chosen: This alternative was developed to address the need for contextual, just-in-time help. It was chosen to test if an interactive, conversational interface could better support users who don't understand government terminology or process requirements.
Usability & Appropriateness:

It is most appropriate for first-time users or those with low digital literacy. Its core usability principle is adaptive guidance—the system can ask clarifying questions and explain why information is needed, which builds user confidence and understanding.

Unique Elements (Not in Other Designs):
  • Conversational, question-and-answer interaction style
  • Adaptive branching based on user responses
  • Just-in-time explanations of government terminology
  • Personalized guidance based on user profile
  • Ability to ask clarifying questions mid-process
Appropriate For:

Users with low digital literacy, complex services with many conditional requirements, scenarios where users need significant hand-holding, services with confusing government terminology.

Design Rationale & Final Selection

Based on user feedback and heuristic evaluation, we selected a hybrid approach combining the best elements of each alternative. Our final prototype uses a wizard-based structure with clear progress indicators for its manageability, supplemented by contextual help modules inspired by the guided assistant for complex sections. This balances the need for clear structure (from the wizard) with the flexibility to provide help (from the assistant), while maintaining a clean, consistent layout superior to the basic form-based approach.

Key Design Decisions:

Structure

Wizard-based with 5-7 logical steps per application

Navigation

Clear progress tracker with ability to jump between completed steps

Help System

Contextual help icons that expand to show explanations and examples

Error Handling

Inline validation with specific, actionable error messages

Final Design Overview

The final design presents a simplified and consistent interface for general government service applications. It introduces a unified layout, clear progress indicators, and structured application steps that guide users from service selection to successful submission.

Key Features Implemented
  • Unified design language across all services
  • Visual progress tracker with completion percentage
  • Contextual help accessible at every step
  • Mobile-first responsive design
  • Clear error messages with correction suggestions
  • Document upload with format guidance
Accessibility Features
  • WCAG 2.1 AA compliant color contrast
  • Keyboard navigation support
  • Screen reader compatibility
  • Adjustable text size
  • Clear focus indicators
  • Alternative text for all images

Evaluation Plan

We developed a comprehensive evaluation plan to assess the prototype against our usability criteria:

Usability Testing

Method: 10 participants completing 5 common tasks

Measures: Success rate, time on task, error count

Goal: Validate efficiency and learnability

Heuristic Evaluation

Method: 3 HCI experts reviewing against Nielsen's heuristics

Measures: Number and severity of violations

Goal: Validate consistency and standards compliance

User Questionnaire

Method: System Usability Scale (SUS) with 15 users

Measures: SUS score (0-100), qualitative feedback

Goal: Validate user satisfaction and perceived usability

Evaluation Results Summary

What Worked Well

  • Step-by-step guidance: 90% of users completed tasks faster compared to original system
  • Consistent layout: Reduced confusion, especially for multi-service applications
  • Progress indicators: Helped users understand their position in complex processes
  • Structured application flow: Error rates reduced by 65% compared to existing system
  • System Usability Score: Achieved 78/100, indicating good to excellent usability
  • Mobile optimization: 95% of mobile users completed tasks successfully

Areas Needing Improvement

  • Document upload requirements: 40% of users needed clarification on file formats
  • Error messages: Needed more specific guidance for correction
  • Contextual help: Some users didn't notice available help options
  • Mobile touch targets: Were too small on some mobile devices
  • Accessibility features: Screen reader compatibility needed improvement
  • Terminology: Some government terms still confused users

User Feedback Summary

User feedback indicated that the step-by-step guidance and consistent layout made the system easier to use. However, some users expressed confusion regarding document upload requirements and suggested clearer instructions and error messages to improve the experience. Positive comments highlighted the reduced cognitive load and clearer navigation paths.

Representative User Quotes:

"The progress tracker gave me confidence that I was moving in the right direction. I didn't feel lost like before."

- Business License Applicant

"I still got confused about which document format to upload. A visual example would help."

Suggested Improvements for Future Work

If more time were available, the design could be improved by adding clearer document upload guidance with visual examples, more informative and actionable error messages, enhanced accessibility features for screen reader users, multilingual support to better serve Kenya's diverse population, and improved mobile optimization with larger touch targets.
