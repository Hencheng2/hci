Initial Problem Statement

Although eCitizen centralizes many government services, users often struggle to complete applications successfully. The system presents long forms, technical language, and inconsistent layouts across services, which increases user errors and frustration. These usability challenges reduce trust in digital government services and discourage adoption.

Target Users & Task Analysis

The primary users of the system are Kenyan citizens with varying levels of digital literacy who access the portal using mobile phones and desktop computers. Secondary users include government officers who review and process submitted applications.

User Tasks Analysis:

  • Selecting a government service from available options
  • Understanding service requirements and prerequisites
  • Filling application forms with personal and service-specific information
  • Uploading required documents in specified formats
  • Submitting applications and making payments if required
  • Tracking application status through the process

Existing System Analysis

The current eCitizen system provides centralized access to government services and reduces the need for physical visits. However, it suffers from inconsistent user interfaces, unclear navigation paths, limited feedback during errors, and poor mobile optimization. These issues significantly affect task completion and overall user satisfaction.

Design Approach

The project follows a user-centered design approach guided by established HCI principles. User needs, task requirements, and usability challenges identified in the problem investigation informed the design decisions. The focus was on simplifying navigation, reducing cognitive load, and providing clear step-by-step guidance throughout service applications.

Design Alternatives Explored

Multiple design alternatives were explored during the design phase, including form-based, wizard-based and Guided Assistant Design. These alternatives helped visualize different navigation structures and interaction flows before selecting a final design that best addressed the identified usability issues.

Form-Based Design Alternative

Rationale: Traditional approach familiar to most users, provides complete form visibility.

Assessment: Users found it overwhelming for complex applications but appreciated seeing all fields.

Wizard-Based Design Alternative

Rationale: Reduces cognitive load by breaking tasks into manageable steps.

Assessment: Users preferred this approach but wanted clearer progress tracking.

Guided Assistant Design Alternative

Rationale: Interactive guidance system that adapts to user needs and skill levels.

Assessment: Highly rated for novice users but considered slow by experienced users.

Evaluation Results

What Worked Well

  • Step-by-step guidance improved task completion
  • Consistent layout reduced confusion
  • Progress indicators helped users understand position
  • Error rates reduced by 65% compared to existing system
  • Achieved System Usability Score of 78/100

Areas Needing Improvement

  • Document upload requirements needed clarification
  • Error messages needed more specific guidance
  • Contextual help wasn't always noticed
  • Mobile touch targets were too small on some devices
  • Screen reader compatibility needed improvement
